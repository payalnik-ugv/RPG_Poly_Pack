Эволюцию Unity Engine условно можно поделить на несколько поколений:
> 2005-2014 г. — впервые игровой движек Unity появился в 2005 г. на базе операционной системы Windows
                 до 2014 г. движек Unity развивался и набирал популярность в игровой индустрии
				 версии Unity Engine вышедших до 2014 г. свободно разпространялись и многие разработчики использовали его на написания 2/3D игр
> 2015-2017 г. — это поколение Unity-5 (уже сформированый инструмент со всеми необходимыми библиотеки и инструментами для написания игр)
                 целые компании начали приобретать коммерческую версию движка для разработки корпоративных проектов
				 в последствии коммерческии версии движка Unity получили завышенную стоимость для исрользования в корпоративных проектов
> 2018-2024 г. — это Unity-LTS
                 позже появились отдельные сообщества, которые стали развивать Unity Engine, доступный для исрользования больштми компаниями в корпоративных проектов
				 в этот период с 2018 по 2024 движек Unity развивался вместе с новыми вышедшими технолоиями (видеокарт и прочее...)
				 за это время появились специальные LTS сбокри, получившие уже долгосрочную поддержку своих версий (функционал которых продолжают поддерживается и сейчас)
				 Архив версий >> https://unity.com/releases/editor/archive

  ---------------------------
  DirectX или OpenGL — это программные интерфейсы (API)
  Unreal — это профессиональный кроссплатформенный игровой движек, был создан в 1995 году компанией Epic Games
  Unity — это профессиональный кроссплатформенный игровой движек, был создан в 2005 году датской компанией Unity Technologies
  CryEngine — это профессиональный игровой движок, разработанный немецкой компанией Crytek
  ---------------------------
  Разработка видеоигр изначально велась с помощью OpenGL API на базе операционной системы DOS.
  Были сложности с установкой игры на систему, потому-что нужно было выполнить много административной работы для подготовки... но видеоигры работали быстро...
  С появлением операционной системы Windows появился DirectX API... для которого необходимо было переписывать и алаптировать видеоигры...
  Первые видеоигры разрабатывались, с помощью OpenGL API, разрабатывались на разных языках программироваия (Java, Python, C, C#)
  В последствии появились игровые движки (Unreal, Unity, CryEngine) разработка которых велась на языке программироваия C#

Assets — это корневая папка проекта (внутри этой папки будут лежать все папки и файлы с исходным кодом проекта)
Scene — это основное окно в редакторе Unity, по умолчанию создается сцена с названием 'SampleScene' и внутри корневой папки проекта будет создана папка 'Scenes' с нашей сценой 'SampleScene'
       (сцены - это уровни в игре)
        Внутри сцены по умолчанию создается первый обьект — Камера — с названием 'Main Camera'
Sprite — это графический обьект, в 2D играх его называют Спрайтами
         бесплатные паки со спрайтами можно скачать с сайта >> https://kenney.nl/assets >> https://kenney.nl/assets/category:2D?search=platformer >> https://kenney.nl/assets/platformer-pack-remastered
		 Переносим пак 'kenney_platformer-pack-remastered' в наш проект в окно Project
		 Выбираем игровой обьект из папки > PNG > Players > 128x256 > Blue > alienBlue_stand и перетаскиваем его в окно нашей сцены
		 * Придаем игровому обьекту (alienBlue_stand) физические свойства > Add Component > Phisycs 2D > Rigidbody 2D
		   теперь у обьекту 'alienBlue_stand' появилось физическое свойство - масса - запустим нашу игру... пришелец упал, значит на него действует сила гравитации
		 Добавляем платформу (пол) из папки > PNG > Ground > Grass > grass и перенесем его под нашим игровым объектом
		 Необходимо обозначить границы нашего обьекта:
		 * Выбираем игровой обьект > 'alienBlue_stand' > Add Component > Phisycs 2D > Capsule Collider 2D
		  (Collider - это такой компонент, который обозначает физтческие рамки для игрового объекта)
		 * Выбираем платформу (пол) > 'grass' > Add Component > Phisycs 2D > Box Collider 2D
		   появилось физическое свойство - границы обьекта - запустим нашу игру... пришелец приземлился на пол

   Но можно изменить его привязку, например к сцене игры > Render Mode > World Space), при этом указав основную камеру игры (Main Camera)
   --------------
   > Canvas > UI > Image
   Canvas > Inspector > Canvas > Render Mode = Word Space  .........  Измените режим Canvas: смените параметр Render Mode с Screen Space - Overlay на World Space
   Canvas > Inspector > Canvas > Render Mode > Event Camera = Main Camera (Camera)  ......  Назначьте камеру: Перетащите вашу главную камеру из Hierarchy в поле Event Camera в настройках Canvas
   --------------
   Image > UI > Text (TextMashPro)
   Text (TMP) > Main Settings > Material Present = Liberation Sans SDF - Outline
